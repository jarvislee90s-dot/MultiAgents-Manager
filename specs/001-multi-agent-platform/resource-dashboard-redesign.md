# 资源看板 UI/UX 重构设计文档

**功能分支**：`001-multi-agent-platform`
**创建日期**：2026-07-06
**状态**：草案
**输入**：用户反馈 — 资源看板（ExtensionList）的视图组织、资源导入、预设组创建等交互需要重新设计

---

## 1. 设计目标

解决当前资源看板的四个核心问题：
1. **视图混乱**：Skill/MCP/Plugin 与按工具的管理面板混排，展示重复
2. **导入不透明**：扫描后资源去向不明，用户无法选择导入哪些
3. **预设组创建困难**：资源列表为空时无法创建，错误提示不精确
4. **原生资源不可见**：工具自带的资源与全局仓库资源混在一起，无法区分

---

## 2. 核心设计决策

### 2.1 双视图切换（按资源 / 按工具）

| 视图 | 展示内容 | 资源来源 |
|------|---------|---------|
| **按资源分类**（默认） | Skills / MCP / Plugins 三个分栏 | 仅全局仓库（`~/.mam/`） |
| **按工具分类** | Claude / Codex / OpenCode 三个卡片 | 全局仓库映射的 + 原生自带的 |

**关键规则**：
- **按资源分类**：只显示已导入全局仓库的资源（`~/.mam/skills/`、`~/.mam/mcp/`、`~/.mam/plugins/`）
- **按工具分类**：显示该工具的**所有资源**，包括全局仓库映射的（正常显示）和原生自带的（灰色标记）
- 原生资源用特殊标签区分（如"原生"、"未导入"），并提供"导入到全局仓库"入口

### 2.2 资源导入流程

**触发位置**：
1. 看板顶部统一"扫描原生资源"按钮
2. 每个工具卡片上的独立"扫描"按钮

**导入流程**：
1. 用户点击"扫描"按钮
2. 后端扫描工具的原始目录（`~/.claude/skills/`、`~/.codex/skills/` 等）
3. 弹窗展示发现的原生资源列表（按工具分组）
4. 用户勾选想要的资源（支持全选/全不选）
5. 点击"导入选中"或"全部导入"
6. 后端复制到全局仓库（`~/.mam/`）
7. 刷新看板，新导入的资源出现在"按资源分类"视图中

### 2.3 预设组设计

**预设组属性**：
- **通用性**：预设组不绑定特定工具，可应用到任意工具
- **资源来源**：从全局仓库资源中选择
- **兼容性标记**：每个资源标记支持的工具（如"支持 Claude、Codex"）

**创建流程**：
1. 用户点击"创建预设组"
2. 输入预设组名称
3. 资源列表按类型分组（Skill/MCP/Plugin），每组可折叠
4. 用户勾选想要的资源
5. 点击"确认创建"

**应用流程**：
1. 用户点击预设组的"应用到 [工具]"按钮
2. 显示兼容性检查报告（哪些资源兼容、哪些不兼容）
3. 用户确认后应用
4. 只安装兼容的资源，不兼容的跳过并提示

### 2.4 资源分配机制

| 资源类型 | 分配操作 | 底层实现 |
|---------|---------|---------|
| **Skill** | 创建映射路径 | `symlink` 从全局仓库 → 工具的 skill 目录 |
| **MCP** | 修改配置文件 | 在工具的 MCP 配置文件中添加/启用该服务器 |
| **Plugin** | 修改配置文件 | 在工具的 plugin 配置文件中添加/启用该插件 |

**取消分配**：
- **Skill**：删除工具的 skill 目录中的 `symlink`
- **MCP**：从工具的 MCP 配置文件中移除该服务器
- **Plugin**：从工具的 plugin 配置文件中移除该插件

## 4. 数据模型变更

### 4.1 ExtensionRecord（扩展资源）

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct ExtensionRecord {
    pub id: String,           // "skill-xxx", "mcp-xxx", "plugin-xxx"
    pub kind: String,         // "skill", "mcp", "plugin"
    pub name: String,
    pub description: Option<String>,
    pub source_path: String,    // 原始路径
    pub source_url: Option<String>,
    pub version: Option<String>,
    pub tags: Option<String>,   // 工具兼容性标记（如 "claude,codex"）
    pub suite: Option<String>,
    pub source_tool: Option<String>, // 来源工具（如 "claude", "codex"）
    pub is_native: bool,        // 是否为原生资源（未导入全局仓库）
}
```

### 4.2 新增表：native_extensions

```sql
CREATE TABLE IF NOT EXISTS native_extensions (
    id          TEXT PRIMARY KEY,
    kind        TEXT NOT NULL,        -- "skill", "mcp", "plugin"
    name        TEXT NOT NULL,
    description TEXT,
    source_path TEXT NOT NULL,        -- 原始路径（如 ~/.claude/skills/skill-name）
    source_tool TEXT NOT NULL,        -- 来源工具（claude/codex/opencode）
    detected_at TEXT NOT NULL,
    imported    INTEGER NOT NULL DEFAULT 0  -- 是否已导入全局仓库
);
```

## 5. API 变更

### 5.1 新增命令

```rust
/// 扫描工具的原生资源（未导入全局仓库的）
#[tauri::command]
pub fn scan_native_resources(tool_id: String) -> Vec<NativeExtensionRecord>;

/// 将原生资源导入全局仓库
#[tauri::command]
pub fn import_native_resources(items: Vec<String>) -> ImportStats;

/// 获取工具的所有资源（全局仓库 + 原生）
#[tauri::command]
pub fn list_tool_resources(tool_id: String) -> ToolResourcesResponse;

/// 检查预设组与工具的兼容性
#[tauri::command]
pub fn check_preset_compatibility(preset_id: String, tool_id: String) -> CompatibilityReport;
```

### 5.2 修改命令

```rust
/// 创建预设组（支持从全局仓库选择资源）
#[tauri::command]
pub fn create_preset(name: String, items: Vec<(String, String)>) -> Result<String, String>;

/// 应用预设组到工具（带兼容性检查）
#[tauri::command]
pub fn apply_preset(preset_id: String, tool_id: String) -> PresetApplyResult;
```

## 7. 与现有 spec 的对比

| 项目 | 现有 spec | 本设计变更 |
|------|----------|-----------|
| 资源视图 | 单一视图，按类型分栏 | 双视图切换（按资源/按工具） |
| 资源来源 | 仅全局仓库 | 全局仓库 + 原生资源（标记区分） |
| 资源导入 | 自动扫描导入，无选择 | 扫描后弹窗，支持部分导入 |
| 预设组资源来源 | 从 extensions DB 选择 | 从全局仓库选择 |
| 预设组工具绑定 | 无绑定 | 通用预设组，应用时选工具 |
| 兼容性检查 | 无 | 应用前显示兼容性报告 |
| MCP 管理 | 各工具独立配置 | 全局统一管理 + 分配到工具 |

## 8. 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 双视图切换增加代码复杂度 | 中 | 将视图逻辑拆分为独立组件，保持单一职责 |
| 原生资源扫描可能耗时 | 低 | 异步扫描，显示 loading 状态 |
| 兼容性检查可能不准确 | 中 | 基于资源 tags 字段判断，允许用户手动覆盖 |
| 数据模型变更影响现有数据 | 低 | 新增字段有默认值，不影响现有数据 |
