## 需要注意的坑点和约束

### 后端

- **`install_to_repo` 路径穿越检查**：第 131 行已移除 `/` 禁止，但保留 `..` 和 `\` 禁止。`name` 含 `/` 时 `repo.join(name)` 直接拼接，由 `create_dir_all` 创建嵌套目录。**没有额外验证**嵌套深度或长度。
- **MCP config 路径格式**：`services::toggle_mcp` 读取 `~/.mam/mcp/<name>.json`（`.json` 后缀直接拼接）。`import_mcp_to_ssot` 现在也写入相同路径。两处已统一。
- **trash 命令依赖**：`disable_skill_for_tool` 使用 macOS `trash` 命令。Linux/Windows 回退到 `remove_link`（直接删除）。如果需要在非 macOS 平台也支持回收站，需要平台判断。
- **DB ext_id 新旧格式并存**：旧的 skill（如 `skill-brainstorming` 顶层）和新的（如 `skill-superpowers/brainstorming` 套件）目前共存。`list_ssot_resources` 直接扫描文件系统，不依赖 ext_id 格式，所以两格式兼容。
- **Claude Code 的 `.codex` vs `.agents` 路径**：已验证 `CodexAdapter.skill_dirs()` 返回 `.agents`。`scan_native_resources` 和 `detect_duplicate_skills` 都已统一使用 `.agents`。
- **`get_all_adapters()` 不存在**：adapter 在 `adapter/mod.rs:get_all_sessions` 中内联创建。如果要消除 4 处重复 adapter 创建，需要先添加 `get_all_adapters()` 到 `adapter/mod.rs`。

### 前端

- **弹窗体验**：两种确认弹窗（符号链接 vs 原生目录）通过 `pending.targetType` 区分。`native` 用 `variant="destructive"` 红色按钮，`symlink` 用 `variant="default"` 蓝色按钮。
- **MCP 添加对话框**：当前用原生 `<input>` 和 `<textarea>`，没有表单验证（除名称和命令非空检查）。
- **`formatSkillName` 工具函数**：在两个组件中重复定义（`ResourceByKindView.tsx` 和 `ResourceByToolView.tsx`）。应提取到共享 util 文件。
- **`list_tool_resources` 返回 JSON**：返回 `serde_json::Value` 而非强类型结构。前端 `ToolResources` 类型在 `extension.ts` 中定义。

### 数据流

- **Skill enabled_tools 判断**（两层）：先查 DB `extension_assignments where enabled=true`，再补查各工具原生 skill 目录中是否存在同名路径。两者并集去重。
- **MCP enabled_tools 判断**（三层）：先扫 `~/.mam/mcp/` 目录获取 SSOT 中的 MCP 名称，再从各工具配置文件提取已配置的 MCP 补入，最后用 DB assignment 修正（enabled=true 补入，enabled=false 移除）。
