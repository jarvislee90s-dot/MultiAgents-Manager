## 四、未覆盖功能的技术方案

### 4.1 APP 形态工具监控（Cursor / VS Code / Codex APP）

**问题：** 这些工具是独立的 GUI 应用，没有像 Claude Code 那样的 hook 系统。

**方案 A：进程监控 + 日志解析（推荐，起点简单）**

```
工作原理:
1. sysinfo 扫描进程，发现 cursor / code / codex 进程
2. 从进程的 --extensions-dir 或用户数据目录找到日志文件
3. 解析日志文件提取会话状态

Cursor: ~/Library/Application Support/Cursor/User/logs/
        → 解码日志中的 AI 请求/响应状态
VS Code: ~/Library/Application Support/Code/User/logs/
         → 通过 extension host 日志判断 AI agent 状态
Codex APP: ~/Library/Application Support/Codex/
           → 日志文件或本地 API 端口探测
```

- 优点：不需要修改目标 APP，纯外部监控
- 缺点：日志格式不稳定，可能随版本变化
- 复杂度：🟡 中等

**方案 B：MCP Server 桥接（扩展性强）**

```
工作原理:
1. 在 Cursor/VS Code 中安装一个轻量 MCP Server 插件
2. MCP Server 暴露会话状态查询接口
3. 我们的看板通过 MCP 协议读取状态

实现:
- VS Code Extension API 提供 onDidChangeActiveTextEditor 等事件
- 插件监听 AI agent 活动并通过 MCP Server 暴露状态
- 看板端作为 MCP Client 轮询或订阅
```

- 优点：标准化（MCP协议），扩展性强
- 缺点：需要用户安装插件
- 复杂度：🟡 中等

**方案 C：本地 HTTP 端口探测（最简单）**

```
工作原理:
1. 检测 Cursor/VS Code 的开发服务器端口
2. 轮询 localhost:端口号/health 或类似端点
3. 根据响应判断 AI agent 是否在运行

Cursor: 通常监听 ~63342 或随机端口
VS Code: 可通过 code --status 获取
```

- 优点：极简，无需安装任何东西
- 缺点：依赖工具暴露端口，覆盖不全
- 复杂度：🟢 低

**推荐路径：** 先用方案 A（进程监控+日志解析）做基础监控，后续通过方案 B（MCP Server 插件）增强精度。

### 4.2 APP 形态工具跳转

**问题：** 无法用 AppleScript 统一控制所有 APP。

**方案：跨平台窗口管理**

```
macOS:
  - AppleScript: tell application "Cursor" to activate
  - open -a "Cursor" /path/to/project
  - 已有现成方案: agent-sessions 的 terminal/mod.rs 模式

Linux:
  - xdotool: xdotool search --name "Cursor" windowactivate
  - wmctrl: wmctrl -a "Cursor"

Windows:
  - Win32 API: SetForegroundWindow
  - PowerShell: (Get-Process Cursor).MainWindowHandle
```

Rust crate 推荐：
- `active-win-pos-rs` — 获取当前活动窗口信息（跨平台）
- `window-vibratio` — 窗口管理
- `enigo` — 模拟键盘/鼠标（用于 send-keys 替代）

**推荐路径：** 封装一个 `WindowManager` trait，macOS 用 AppleScript，Linux 用 xdotool，Windows 用 Win32 API，与终端跳转统一接口。

### 4.3 子 Agent 级 skill 分配

**问题：** 多 Agent 工具（如 Hermes、OpenCode）内部有多个 Agent，需要为每个子 Agent 分配不同 skill。

**方案：扩展 AgentAdapter，增加 SubAgent 概念**

```rust
// 在 HarnessKit 的 AgentAdapter 基础上扩展
pub trait AgentAdapter: Send + Sync {
    // ... 现有方法 ...

    /// 返回此 agent 的子 agent 列表（如 Hermes 的多个 agent persona）
    fn sub_agents(&self) -> Vec<SubAgent> {
        vec![] // 默认无子 agent
    }

    /// 子 agent 的 skill 目录
    fn sub_agent_skill_dirs(&self, sub_agent_id: &str) -> Vec<PathBuf> {
        vec![]
    }
}

pub struct SubAgent {
    pub id: String,
    pub name: String,
    pub config_path: PathBuf,  // 如 Hermes 的 agents/<name>.yaml
    pub skill_base_dir: PathBuf,
}
```

各工具的子 Agent 结构：
- **Hermes**: `~/.hermes/agents/<name>.yaml`，每个 agent 有独立 skill 目录
- **OpenCode**: `~/.opencode/agents/<name>/`，每个 agent 可配独立 skill
- **Claude Code subagents**: `~/.claude/agents/<name>.md`，通过 agent 定义文件指定

**篮子式分配模型：**

```
全局仓库 (~/.mam/skills/)
├── skill-a/
├── skill-b/
├── ...100 个 skill

映射配置 (SQLite):
┌─────────────┬──────────┬────────────┬─────────┐
│ skill_id    │ agent_id │ sub_agent  │ enabled │
├─────────────┼──────────┼────────────┼─────────┤
│ skill-a     │ codex    │ (global)   │ true    │
│ skill-a     │ claude   │ (global)   │ true    │
│ skill-b     │ codex    │ (global)   │ false   │
│ skill-c     │ hermes   │ researcher │ true    │
│ skill-c     │ hermes   │ coder      │ false   │
└─────────────┴──────────┴────────────┴─────────┘

篮子 (Presets):
┌──────────────┬─────────────────────────┐
│ preset_name  │ skills                   │
├──────────────┼─────────────────────────┤
│ 前端开发      │ skill-a, skill-b, skill-c│
│ 后端开发      │ skill-d, skill-e         │
│ 全栈          │ skill-a, skill-d, skill-f│
└──────────────┴─────────────────────────┘

应用篮子到 (agent, sub_agent):
"前端开发" → (codex, global)    → 启用 a,b,c
"前端开发" → (hermes, coder)    → 启用 a,b,c
"后端开发" → (hermes, researcher)→ 启用 d,e
```

**推荐路径：** 先实现全局仓 + symlink + 按工具开关（直接移植 skills-manager-jw），再加 Presets（移植 xingkongliang），最后扩展 sub_agent 维度。

### 4.4 统一红绿灯悬浮窗

**方案：Tauri 系统托盘 + 悬浮窗**

```
技术方案:
1. Tauri tray-icon: 系统托盘显示聚合状态（红/黄/绿）
   - 扫描所有活跃会话
   - 任一 waiting → 红灯
   - 全部 idle/finished → 绿灯
   - 有 working → 黄灯
   - 点击托盘 → 展开悬浮窗

2. 悬浮窗 (always-on-top window):
   - Tauri window with decorations=false, always_on_top=true
   - 贴边显示（参考 agent-manager-x 的 mini sidebar）
   - 列出所有会话卡片，红/黄/绿状态灯
   - 点击卡片 → 跳转对应终端/APP

3. 全局快捷键:
   - tauri-plugin-global-shortcut
   - 切换悬浮窗显示/隐藏
```

参考项目：
- agent-manager-x 的悬浮窗 UX（贴边、hover 展开、热键切换）
- cc-agent-watch 的六边形状态聚合（多个会话合并为一个指示灯）

