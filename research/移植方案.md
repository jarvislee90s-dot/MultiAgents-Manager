# 移植方案：从现有参考项目到我们的项目

## 核心结论

**不需要 fork 某一个项目，而是以 tauri-app-template 为脚手架，移植 agent-sessions 的核心模块。**

agent-sessions 只有 3280 行 Rust + 1700 行 TS/TSX，但已经包含了我们阶段一需要的全部核心功能：AgentDetector trait、进程发现、JSONL 解析、状态判断、终端跳转、系统托盘。而 tauri-app-template 提供了更完整的基础设施（i18n、主题、更新、标题栏）。

## 项目对照

### 脚手架：tauri-app-template（直接 copy + rename）

已有基础设施，开箱即用：

| 已有功能 | 文件 | 说明 |
|---------|------|------|
| Tauri 2 + React 19 + Vite 7 | 整个项目 | 技术栈完全匹配 |
| shadcn/ui 组件 | src/components/ui/ | button, card, dialog, dropdown-menu, input, progress, sonner |
| 暗色/亮色主题 | src/components/theme-provider.tsx, mode-toggle.tsx | next-themes |
| 国际化（中/英） | src/i18n/ | i18next，已预置 zh.json 和 en.json |
| 自动更新 | src/hooks/use-updater.ts, updater-dialog.tsx | tauri-plugin-updater |
| 自定义标题栏 | src/components/title-bar.tsx, window-frame.tsx | 无边框窗口 |
| 全局快捷键设置 | src/components/shortcut-input.tsx, src/lib/shortcut.ts | tauri-plugin-global-shortcut |
| 单实例 | src-tauri/Cargo.toml | tauri-plugin-single-instance |

**操作：** 直接 copy 整个 tauri-app-template 目录，rename crate 名称为 `multi-agents-manager`。

### 监控模块：agent-sessions（移植核心 Rust 模块）

agent-sessions 的 Rust 后端结构清晰，模块间耦合低，可以整体移植：

| 模块 | 文件 | 行数 | 移植方式 |
|------|------|------|---------|
| AgentDetector trait | agent/mod.rs | 108 | 直接复制，改名为 AgentAdapter |
| Claude 检测器 | agent/claude.rs | ~30 | 直接复制 |
| OpenCode 检测器 | agent/opencode.rs | 489 | 直接复制（阶段三用） |
| 进程发现 | process/claude.rs | 164 | 直接复制 + 新增 codex.rs |
| 数据模型 | session/model.rs | 72 | 直接复制 + 增加 Codex 类型 |
| 状态判断 | session/status.rs | 192 | 直接复制 |
| JSONL 解析 | session/parser.rs | 690 | 复制 Claude 解析器，新增 Codex 解析器 |
| 终端跳转 | terminal/ | ~300 | 直接复制（AppleScript + iTerm + Terminal.app + tmux） |
| Tauri 命令 | commands/handlers.rs | 114 | 直接复制 |
| 应用入口+托盘 | lib.rs | 106 | 合并到 template 的 lib.rs（template 的托盘更完整） |

**需要新增的：**
- `process/codex.rs` — Codex CLI 进程发现（参考 process/claude.rs 模式，进程名改为 `codex`）
- `session/codex_parser.rs` — Codex JSONL 解析器（适配 `type` + `payload` 结构，参考接口文档）
- `agent/codex.rs` — Codex 检测器

**需要修改的：**
- `session/model.rs` — AgentType 增加 `Codex` 变体
- `agent/mod.rs` — detectors 列表增加 CodexDetector

## 阶段一具体操作步骤

### Step 1: 脚手架（半天）

```bash
# 1. 复制 tauri-app-template
cp -r reference/tauri-app-template/ src/

# 2. 重命名 crate
# src-tauri/Cargo.toml: name = "multi-agents-manager"
# src-tauri/src/lib.rs: rename lib name

# 3. 合并依赖
# 从 agent-sessions 的 Cargo.toml 加入: sysinfo, dirs, chrono, once_cell
# 从宪法加入: rusqlite, dashmap, notify, fs2

# 4. 验证: cargo build + pnpm dev
```

### Step 2: 移植监控模块（2 天）

```bash
# 1. 复制 agent-sessions 的 Rust 模块
cp -r reference/agent-sessions/src-tauri/src/agent/ src/src-tauri/src/adapter/
cp -r reference/agent-sessions/src-tauri/src/process/ src/src-tauri/src/process/
cp -r reference/agent-sessions/src-tauri/src/session/ src/src-tauri/src/session/
cp -r reference/agent-sessions/src-tauri/src/terminal/ src/src-tauri/src/terminal/

# 2. 修改 module 名和导入路径
# 3. 合并 lib.rs 的托盘逻辑
# 4. 验证: cargo test
```

### Step 3: 新增 Codex adapter（1 天）

```rust
// process/codex.rs — 参考 process/claude.rs，进程名改为 "codex"
// session/codex_parser.rs — 适配 type+payload 结构
// agent/codex.rs — CodexDetector
// session/model.rs — AgentType::Codex
```

### Step 4: 移植前端组件（1 天）

```bash
# 复制 agent-sessions 的前端组件
cp reference/agent-sessions/src/components/SessionCard.tsx src/components/
cp reference/agent-sessions/src/components/SessionGrid.tsx src/components/
cp reference/agent-sessions/src/hooks/useSessions.ts src/hooks/
cp reference/agent-sessions/src/types/session.ts src/types/
cp reference/agent-sessions/src/lib/formatters.ts src/lib/
```

### Step 5: 通知系统（1 天）

```bash
# 从 claude-control 移植通知逻辑
# useNotificationSound.ts → 前端直接复用
# useDesktopNotification.ts → 前端复用或改用 tauri-plugin-notification
# Hook 注册器 → 改写为 Rust
```

### Step 6: 看板 UI + 红绿灯（2 天）

```
基于 tauri-app-template 的 shadcn/ui 组件 + agent-sessions 的 SessionCard
新增: 红绿灯状态指示器组件
新增: 托盘图标状态更新（红/黄/绿）
```
